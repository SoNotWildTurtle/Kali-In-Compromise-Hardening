# NN IDS Audit Gate

`nn_ids_audit_gate.py` turns the NN IDS model audit report into an explicit defensive policy decision. The previous audit module measures class balance, balanced accuracy, F1, drift, robustness, and explainability drift; this gate decides whether the currently deployed model is acceptable, should be watched, should be retrained, or should be restored from the latest known-good snapshot.

## Why this exists

Current intrusion-detection ML research continues to show three operational risks that matter more than raw training accuracy:

1. **Concept drift and class imbalance** can silently reduce detection quality as traffic changes and rare attacks remain underrepresented.
2. **Adversarial perturbations** can degrade model confidence and detection accuracy even when clean-data metrics look good.
3. **Explainability drift** can show that the model is relying on different features than expected, which may signal poisoning, environment change, or brittle learning.

The gate is intentionally conservative. It does not automatically replace a model unless the operator explicitly enables auto actions with `NN_IDS_GATE_AUTO_ACTIONS=1`. By default, it writes a decision report to `/var/log/nn_ids_audit_gate.json` and keeps the next action reviewable.

## Inputs

Default input paths:

- Audit report: `/var/log/nn_ids_model_audit.json`
- Gate state: `/opt/nnids/audit/audit_gate_state.json`
- Gate report: `/var/log/nn_ids_audit_gate.json`

The gate consumes the JSON generated by `nn_ids_model_audit.py`.

## Decisions

| Decision | Meaning | Default action |
| --- | --- | --- |
| `accept` | Metrics and drift are within policy. | No action. |
| `watch` | Medium-risk degradation exists. | Log and wait for more audits. |
| `retrain` | F1 or balanced accuracy fell below policy. | Recommend/start retrain service. |
| `restore` | Audit is missing/failed, or repeated retrains did not recover. | Recommend/start restore service. |

## Thresholds

Tune with environment variables or a systemd drop-in:

```bash
NN_IDS_GATE_MIN_F1=0.70
NN_IDS_GATE_MIN_BALANCED_ACCURACY=0.70
NN_IDS_GATE_MIN_ROBUSTNESS_INDEX=0.65
NN_IDS_GATE_MAX_MEAN_Z=3.0
NN_IDS_GATE_MAX_SHIFTED_FEATURES=10
NN_IDS_GATE_MAX_CONSECUTIVE_RETRAIN=2
NN_IDS_GATE_AUTO_ACTIONS=0
```

`NN_IDS_GATE_AUTO_ACTIONS=0` is the secure default. Set it to `1` only after validating the retrain and restore services in the target VM.

## Manual usage

```bash
sudo /usr/local/bin/nn_ids_model_audit.py
sudo /usr/local/bin/nn_ids_audit_gate.py
sudo cat /var/log/nn_ids_audit_gate.json
```

## Systemd usage

```bash
sudo systemctl enable --now nn_ids_audit_gate.timer
sudo systemctl start nn_ids_audit_gate.service
sudo systemctl status nn_ids_audit_gate.service
```

## Safety model

- Reads local audit files only.
- Writes local state and log files only.
- Does not scan networks.
- Does not modify firewall policy.
- Does not contact remote services.
- Does not perform offensive testing.
- Auto-starts retrain/restore only when explicitly enabled.
