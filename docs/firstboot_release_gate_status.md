# Firstboot release gate status reader

`firstboot_release_gate_status.py` is a passive operator helper for reading the aggregate-only shell summary emitted by `firstboot_release_gate.py --summary`.

The JSON and Markdown release-gate artifacts remain authoritative. This reader exists for dashboards, smoke checks, shift handoffs, and recovery runbooks that need a low-friction pass/defer answer without parsing the full JSON report or sourcing a shell file.

## Usage

```bash
python3 firstboot_release_gate_status.py \
  --summary /var/log/firstboot_release_gate.summary.env \
  --format text
```

For release gates and firstboot promotion checks, require a passing aggregate summary:

```bash
python3 firstboot_release_gate_status.py \
  --summary /var/log/firstboot_release_gate.summary.env \
  --format json \
  --require-pass
```

The command exits `0` when the summary is valid and passing. With `--require-pass`, it exits `7` when the summary is missing, malformed, deferred, stale/skewed, privacy-scope incompatible, or otherwise not passing.

## Output contract

The helper emits either text or JSON with:

- `decision` and `release_gate` for the reader result.
- Source decision/status/timestamp fields copied from the aggregate summary.
- Artifact, blocker, and stale/skewed counts.
- Validation blockers for malformed or incomplete summaries.
- Privacy-safe operator next steps.

The reader never embeds raw logs, packets, captures, credentials, hostnames, usernames, secrets, model binaries, datasets, environment identifiers, or remediation prose from the full gate. It parses quoted key/value lines itself and does not `source` the `.env` file.

## Threat-model rationale

Operations dashboards and handoff scripts often need a simple release status but should not parse larger reports, ingest raw telemetry, or execute shell content. This helper keeps that path least-privilege and read-only by validating a strict aggregate contract, rejecting unexpected privacy scopes, and refusing to treat malformed summaries as passing.

The helper does not open network sockets, execute host commands, restart services, alter firewall rules, modify firstboot state, approve restores, or update NN IDS models/datasets.

## Compatibility

The helper uses only the Python standard library and is safe for Kali/Debian Python 3 environments. It reads the summary generated by `firstboot_release_gate.py` and does not require systemd, root privileges, external packages, or network access.

## Rollback

Rollback is additive and low risk:

1. Stop invoking `firstboot_release_gate_status.py` from dashboards, smoke checks, or release scripts.
2. Continue using the authoritative JSON and Markdown artifacts directly.
3. Remove the helper from ISO packaging if desired.
4. Revert this helper, tests, docs, packaging entry, and changelog entry.

No upstream firstboot manifest, NN IDS model card, JSON release gate, Markdown release gate, summary file, host setting, VM setting, firewall rule, service state, approval, restore state, model file, or dataset is modified by this helper.
