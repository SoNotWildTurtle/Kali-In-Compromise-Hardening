# Firstboot release-gate operator digest

`firstboot_release_gate_operator_digest.py` creates a passive, privacy-safe handoff summary from the existing firstboot release-gate status JSON and bundle manifest JSON. It is intended for release review, recovery bundles, shift handoffs, and operator dashboards that need a concise manager-readable decision without reading sensitive source material.

## Inputs

Default paths:

- `/var/log/firstboot_release_gate.status.json`
- `/var/log/firstboot_release_gate.bundle_manifest.json`

Both inputs are aggregate artifacts. The digest does not inspect raw logs, packets, captures, credentials, hostnames, usernames, secrets, model binaries, datasets, environment identifiers, live services, firewall rules, or host/VM settings.

## Usage

Generate JSON:

```bash
/usr/local/bin/firstboot_release_gate_operator_digest.py \
  --status-json /var/log/firstboot_release_gate.status.json \
  --bundle-json /var/log/firstboot_release_gate.bundle_manifest.json \
  --output /var/log/firstboot_release_gate.operator_digest.json
```

Generate Markdown for a handoff packet:

```bash
/usr/local/bin/firstboot_release_gate_operator_digest.py \
  --status-json /var/log/firstboot_release_gate.status.json \
  --bundle-json /var/log/firstboot_release_gate.bundle_manifest.json \
  --output /var/log/firstboot_release_gate.operator_digest.md \
  --format markdown
```

Fail closed for release gates:

```bash
/usr/local/bin/firstboot_release_gate_operator_digest.py \
  --require-pass \
  --output /var/log/firstboot_release_gate.operator_digest.json
```

`--require-pass` exits with code `7` unless the status and bundle are both passing, internally consistent, free of blockers, and all required bundle artifacts are present with SHA-256 references.

## Output contract

The JSON digest includes:

- `schema_version` and `component` for contract checks.
- `decision` and `release_gate` for release automation.
- `source_paths` for traceability.
- `source_summary` with aggregate decision state, timestamp references, blocker counts, stale/skew counts, and bundle artifact counts.
- `blockers` for machine-readable remediation.
- `manager_summary` for human review.
- `handoff_checklist` for release and recovery workflows.
- `privacy_scope`, `privacy_exclusions`, `safe_default`, and `rollback_note` for security review.

## Security and privacy rationale

This helper is additive and read-only. It only parses the aggregate status and bundle manifest JSON generated by the existing passive firstboot release-gate chain. It never sources shell content, opens sockets, restarts services, changes firewall rules, modifies models or datasets, approves restores, or mutates host/VM state.

The digest deliberately summarizes only the existence, consistency, aggregate decision state, and hash coverage of release evidence. This keeps review artifacts useful for managers and operators while avoiding sensitive operational telemetry.

## Compatibility

The JSON output is the stable automation interface. Markdown is intended for readable release packets and shift handoffs. Existing status, bundle, firstboot, and release-gate artifacts remain unchanged.

## Rollback

Delete generated files:

```bash
sudo rm -f /var/log/firstboot_release_gate.operator_digest.json
sudo rm -f /var/log/firstboot_release_gate.operator_digest.md
```

To disable automatic refresh, remove the digest `ExecStartPost` lines from `firstboot_release_gate.service` and run:

```bash
sudo systemctl daemon-reload
sudo systemctl restart firstboot_release_gate.service
```

Rollback does not affect the upstream firstboot release-gate JSON, Markdown, summary, status, or bundle manifest artifacts.
