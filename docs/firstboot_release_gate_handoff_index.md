# Firstboot release-gate handoff index

`firstboot_release_gate_handoff_index.py` is a passive helper for release review, firstboot handoff, and recovery bundle promotion. It indexes the existing firstboot release-gate JSON/Markdown artifacts, records file sizes and SHA-256 hashes, checks aggregate decisions for consistency, and emits one operator-friendly JSON or Markdown index.

## Usage

```bash
firstboot_release_gate_handoff_index.py \
  --output /var/log/firstboot_release_gate.handoff_index.json

firstboot_release_gate_handoff_index.py \
  --format markdown \
  --output /var/log/firstboot_release_gate.handoff_index.md

firstboot_release_gate_handoff_index.py \
  --require-ready \
  --output /var/log/firstboot_release_gate.handoff_index.json
```

The default input paths are the artifacts generated by `firstboot_release_gate.service`:

- `/var/log/firstboot_release_gate.json`
- `/var/log/firstboot_release_gate.md`
- `/var/log/firstboot_release_gate.summary.env`
- `/var/log/firstboot_release_gate.status.json`
- `/var/log/firstboot_release_gate.bundle_manifest.json`
- `/var/log/firstboot_release_gate.bundle_manifest.md`
- `/var/log/firstboot_release_gate.operator_digest.json`
- `/var/log/firstboot_release_gate.operator_digest.md`

## Output contract

The JSON output contains:

- `schema_version` and `component` for machine-readable consumers.
- `decision` and `release_gate`, using `approved/pass` only when required JSON artifacts exist, parse, pass, and agree.
- `artifact_counts` with total, required, present, Markdown, and hashed artifact counts.
- `artifacts` with label, path, required flag, format, presence, size, SHA-256, and known aggregate decision fields.
- `blockers` for missing required artifacts, invalid JSON, component mismatches, failing release gates, or inconsistent aggregate decisions.
- `handoff_checklist`, `privacy_scope`, `privacy_exclusions`, `safe_default`, and `rollback_note`.

The Markdown output renders the same information for manager or operator handoff without requiring JSON tooling.

## Threat-model rationale

The helper is intentionally read-only and aggregate-only. It does not inspect raw logs, packets, captures, credentials, hostnames, usernames, secrets, model binaries, datasets, or environment identifiers. It hashes files already produced by the firstboot release-gate workflow so operators can verify that handoff bundles are complete and unchanged.

The helper is fail-closed when required status, bundle manifest, or operator digest JSON artifacts are missing, malformed, failing, or inconsistent. With `--require-ready`, it exits non-zero when the handoff index is not approved.

## Service integration

`firstboot_release_gate.service` refreshes JSON and Markdown handoff indexes after status, bundle manifest, and operator digest artifacts are generated. The service remains passive and sandboxed; the index helper only reads the expected `/var/log` artifacts and writes derived indexes back to `/var/log`.

## Compatibility

This is additive. Existing firstboot release-gate outputs, service behavior, JSON schemas, Markdown summaries, and release-gate decisions remain compatible. Consumers that do not need the handoff index can ignore the new files.

## Rollback

Remove `/var/log/firstboot_release_gate.handoff_index.json` and `/var/log/firstboot_release_gate.handoff_index.md`, or revert the helper, service post-steps, packaging entry, docs, tests, and changelog lines. Upstream release-gate, status, bundle manifest, and operator digest artifacts remain unchanged.
